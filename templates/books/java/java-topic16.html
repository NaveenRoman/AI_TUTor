Q1: What is JVM, JRE, and JDK?
JVM executes bytecode; JRE provides JVM + libraries; JDK contains JRE + development tools (javac, jar).
Q2: What is classpath?
Classpath tells JVM where to find user classes and libraries (directories/jar files).
Q3: What does 'public static void main(String[] args)' mean?
'public' accessible; 'static' JVM calls without instance; 'void' returns nothing; 'main' entry point; String[] args holds CLI args.
Q4: Difference between == and equals()?
== compares reference identity for objects; equals() compares logical equality (can be overridden).
Q5: What is garbage collection?
Automatic memory management removing objects with no live references; triggered by JVM GC algorithms.
Q6: What is final keyword usage?
final variable (constant), final method (cannot override), final class (cannot extend).
Q7: What is finally block?
finally executes after try/catch regardless of exceptions; used for cleanup (may be skipped on System.exit()).
Q8: What is transient?
Marks fields not to be serialized by default Java serialization.
Q9: What is volatile?
Ensures reads/writes go to main memory ‚Äî prevents thread caching; provides visibility but not atomicity.
Q10: What are primitive types in Java?
byte, short, int, long, float, double, char, boolean ‚Äî stored by value.
Q11: Autoboxing & unboxing?
Automatic conversion between primitives and wrapper classes (int ‚Üî Integer).
Q12: What is String immutability? Benefits?
String instances cannot be changed; helps security, caching, thread-safety; String pool reduces duplication.
Q13: Difference between String, StringBuilder, StringBuffer?
String immutable; StringBuilder mutable & not synchronized (faster); StringBuffer synchronized (thread-safe, slower).
Q14: What is package-private (default) access?
Accessible within same package only (no modifier).
Q15: What is an interface vs abstract class?
Interface defines contract (Java 8+ can have default/static methods); abstract class can contain state and implemented methods; choose based on extends vs implements and shared state.
Q16: What is method overloading vs overriding?
Overloading: same method name different params in same class; Overriding: subclass redefines parent method signature.
Q17: What is polymorphism?
Ability for objects to take many forms; runtime polymorphism via method overriding and dynamic dispatch.
Q18: What is encapsulation?
Hiding internal data and providing access via methods (getters/setters) to protect invariants.
Q19: What is inheritance?
Mechanism where a class (subclass) acquires properties and behavior from another (superclass).
Q20: What are wrapper classes?
Object representations of primitives: Integer, Double, Boolean, etc.; provide utility methods.
Q21: What is Comparable vs Comparator?
Comparable defines natural order via compareTo; Comparator defines external ordering via compare(a,b).
Q22: What are collections frameworks core interfaces?
Collection, List, Set, Queue, Deque, Map (Map not extend Collection).
Q23: ArrayList vs LinkedList?
ArrayList: random access O(1), insert/remove O(n); LinkedList: O(1) insert/remove at ends, slower random access.
Q24: HashMap vs TreeMap?
HashMap is unordered, O(1) average lookup; TreeMap is sorted (Red-Black tree), O(log n).
Q25: ConcurrentHashMap advantages?
Thread-safe map with better concurrency than synchronized map; uses internal segmentation/locks (implementation varies by JDK).
Q26: What is Iterator vs ListIterator?
Iterator allows forward-only traversal; ListIterator supports bidirectional traversal and element modification.
Q27: What is fail-fast vs fail-safe iterators?
Fail-fast detect concurrent modification and throw ConcurrentModificationException (e.g., ArrayList). Fail-safe (e.g., CopyOnWriteArrayList) iterate on a snapshot.
Q28: What is synchronization keyword?
Used to control access to blocks/methods by one thread at a time on a monitor (object/class lock).
Q29: What is Thread vs Runnable?
Thread is a thread of execution; Runnable provides run() and is passed to Thread; prefer Runnable for decoupling.
Q30: What is ExecutorService?
Framework to manage thread pools and task execution; use Executors factory methods for pools.
Q31: Explain volatile vs synchronized.
volatile ensures visibility across threads; synchronized ensures mutual exclusion and visibility (enter/exit monitor flushes caches).
Q32: What is deadlock?
When two or more threads wait indefinitely for locks held by each other; avoid by lock ordering or timeouts.
Q33: What is race condition?
Incorrect behavior due to unsynchronized access to shared mutable data; fix with synchronization/atomic operations.
Q34: What are atomic classes?
java.util.concurrent.atomic package classes (AtomicInteger, AtomicReference) for lock-free thread-safe operations.
Q35: What is ThreadLocal?
Provides thread-confined variables; each thread has its own copy of the variable.
Q36: What is checked vs unchecked exception?
Checked exceptions must be declared/handled (IOException); unchecked (RuntimeException) need not be. Error indicates serious JVM problems.
Q37: When to use custom exceptions?
When you need clearer domain-specific error handling and typed exceptions for callers to react appropriately.
Q38: What is try-with-resources?
Language construct that auto-closes Closeable/AutoCloseable resources at the end of the try block.
Q39: Serialization basics?
Convert object to byte stream via ObjectOutputStream; class must implement Serializable; use serialVersionUID for versioning.
Q40: What is transient and serialVersionUID?
transient excludes fields from serialization; serialVersionUID identifies class version for deserialization compatibility.
Q41: What is reflection?
API to inspect and manipulate classes/methods/fields at runtime (java.lang.reflect). Useful for frameworks; expensive and can break encapsulation.
Q42: What are annotations?
Metadata in code (@Override, @Deprecated, custom annotations); used by compilers and frameworks (DI, JPA).
Q43: What is generics?
Type parameterization enabling compile-time type safety for collections and APIs (List), implemented via type erasure.
Q44: What is type erasure?
Generics are implemented at compile time; runtime doesn't retain generic type info, which can cause some limitations (no new T[]).
Q45: What are wildcards ? extends vs ? super?
? extends T: producer (read); ? super T: consumer (write) ‚Äî PECS (Producer Extends, Consumer Super).
Q46: What is Optional?
Container that may hold a non-null value; helps avoid null checks and NullPointerException; use Optional as return types, not fields.
Q47: What are streams?
Functional-style pipeline for collections: map/filter/reduce; can be sequential or parallel; lazy evaluation.
Q48: When to use parallel streams?
For CPU-bound, large datasets where operations are independent; test and measure ‚Äî overhead can outweigh benefits for small data.
Q49: What is lambda expression?
Anonymous function used with functional interfaces; concise replacing single-method anonymous classes.
Q50: What is method reference?
Shorthand for lambda calling an existing method, e.g., System.out::println or String::toUpperCase.
Q51: What is Optional.map vs flatMap?
map transforms value; flatMap transforms value into Optional and flattens nesting (Optional>).
Q52: What is functional interface?
Interface with single abstract method (SAM); can be used as lambda target (Predicate, Function, Consumer).
Q53: What is default method in interface?
Java 8 feature: interfaces can provide default method implementations to avoid breaking existing code.
Q54: What is module system (Java 9)?
JPMS: modules with module-info.java to declare module dependencies and exported packages, improving encapsulation and startup.
Q55: What is var (local variable type inference)?
Java 10 introduced var for local type inference ‚Äî compiler infers type; still statically typed.
Q56: What are records (Java 14+)?
Compact syntax for immutable data carriers ‚Äî auto-generated constructors, equals, hashCode, toString.
Q57: What are sealed classes (Java 17)?
Restrict which other classes may extend a class using 'permits' ‚Äî improving exhaustive checks and security.
Q58: What is switch expression (Java 14)?
Enhanced switch can return values, use 'case L ->' syntax and avoids fall-through by default.
Q59: What is String.intern()?
Returns canonical representation from String pool; useful for memory savings but use cautiously.
Q60: What is ClassLoader?
Loads classes into JVM (bootstrap, platform, system/app); custom CLs used in complex apps (containers, plugins).
Q61: What is reflection performance/security trade-off?
Reflection is slower and can bypass encapsulation; use for frameworks; avoid in hot loops and sensitive logic.
Q62: How does equals() and hashCode() contract work?
If equals() two objects are equal, their hashCode() must be equal. Important for hash-based collections.
Q63: What is null pointer exception and how to avoid?
NPE is thrown when accessing member on null reference; avoid with checks, Optional, or defensive coding.
Q64: How to create immutable class?
Make class final, fields private final, no setters, defensive copies for mutable fields, provide constructors only.
Q65: What is observer pattern?
Behavioral pattern where observers register to receive updates from subject (publish/subscribe model).
Q66: What is singleton enum pattern?
Use enum with single element to implement singleton ‚Äî handles serialization and thread-safety automatically.
Q67: What is memory leak in Java?
Objects still referenced but not used; common causes: static collections, listeners not removed, caches without eviction.
Q68: What is finalizer vs try-with-resources?
finalize() is unreliable and deprecated; use AutoCloseable and try-with-resources for deterministic cleanup.
Q69: How to sort custom objects?
Implement Comparable or provide Comparator to Collections.sort/Stream.sorted.
Q70: What is binarySearch requirement?
Collection must be sorted according to the comparator/natural order before binarySearch; otherwise result undefined.
Q71: What is reflection? (duplicate intentional for practice)
See Q41 concise explanation: runtime inspection and manipulation of classes, methods, fields.
Q72: What is dynamic proxy?
Proxy generated at runtime that implements interfaces and delegates method calls ‚Äî used for AOP, logging, transaction management.
Q73: What is default serial form?
ObjectOutputStream default serializes non-transient fields; customize with writeObject/readObject methods.
Q74: What is instanceof vs isInstance?
instanceof checks at compile-time against a type; Class.isInstance allows dynamic class checks.
Q75: How to create a thread-safe singleton?
Use enum-singleton, eager initialization, or double-checked locking with volatile instance.
Q76: How to stop a thread?
Use interruption (thread.interrupt()) and cooperative checks (Thread.currentThread().isInterrupted()), avoid deprecated stop().
Q77: What is sleep vs wait?
sleep() pauses thread without releasing monitor; wait() releases monitor and waits for notify()/notifyAll().
Q78: Difference between notify and notifyAll?
notify wakes one waiting thread; notifyAll wakes all waiting threads for the monitor.
Q79: How to implement producer-consumer?
Use BlockingQueue (preferred) or wait/notify with synchronized buffers; BlockingQueue handles concurrency safely.
Q80: What is CompletableFuture?
Promise-like API for async programming with chaining, combinators, and completion stages (non-blocking).
Q81: Where to use synchronized collections?
Use Collections.synchronizedList for simple synchronization; prefer concurrent collections (ConcurrentHashMap) for better scalability.
Q82: What is memory model (JMM)?
Java Memory Model defines visibility, ordering, and atomicity rules across threads; volatile/synchronized establish happens-before relationships.
Q83: What are design patterns categories?
Creational, Structural, Behavioral (e.g., Singleton, Factory, Adapter, Strategy, Observer).
Q84: What is adapter pattern?
Converts interface of a class into another interface clients expect; useful to integrate incompatible APIs.
Q85: What is decorator pattern?
Attach responsibilities to objects dynamically via wrapping classes; alternative to subclassing for flexible behavior.
Q86: What is strategy pattern?
Define a family of algorithms, encapsulate each, and make them interchangeable (via interfaces).
Q87: What is template method?
Define skeleton of an algorithm in base class, delegate steps to subclasses (inversion of control).
Q88: How to debug NullPointerException?
Read stack trace, locate offending line, add null checks or assert invariants earlier, use Optional for returns.
Q89: What is stack vs heap?
Stack: method frames, local primitives/references; Heap: objects and arrays managed by GC.
Q90: How to profile a Java app?
Use tools like VisualVM, JFR (Flight Recorder), async profiler, and heap dump analyzers.
Q91: What is JIT compiler?
Just-In-Time compiler optimizes bytecode to native code at runtime for hotspots improving performance.
Q92: Explain class loading delegation?
Parent-first delegation model: child ClassLoader delegates to parent to avoid duplicate class definitions in JVM.
Q93: What is soft/weak/phantom references?
Reference types with different GC strengths used for caches (Soft), canonical mappings (Weak), and post-mortem cleanup (Phantom).
Q94: How to implement equals safely?
Check reference equality, instanceof, cast, and compare significant fields; override hashCode consistent with equals.
Q95: What is contract of hashCode?
Consistent during an object life (unless fields used change), equal objects have equal hash code, not required unique per object.
Q96: What are metrics to show code quality?
Code coverage, cyclomatic complexity, code smells, static analysis warnings, test pass rate, performance benchmarks.
Q97: What is hotspot analysis?
Identify frequently executed code paths (hotspots) to optimize via profiling and tuning (JIT or algorithmic change).
Q98: When to use microbenchmarks?
Measure performance of small code sections with JMH to avoid JVM warmup and inaccurate measurements.
Q99: How to handle large files?
Use streaming APIs (Files.lines, Buffered streams), process chunks, avoid loading entire file into memory.
Q100: How to prepare for viva quickly?
Practice short 30-second explanations, mock oral rounds, and concise diagrammatic answers for core topics.
Q101: What is defensive programming?
Write code anticipating incorrect usage, with validation, preconditions, and helpful exceptions.
Q102: How to write production-grade Java?
Follow SOLID, proper logging, metrics, proper exception handling, tests, CI, and monitoring in production.

üß© OOP Principles (Beginner ‚Üí Pro)
  1. Single Responsibility: keep classes focused; example: separate DTO, service, DAO.
  2. Open/Closed: extend via interfaces/strategy; use composition over inheritance.
  3. Liskov: subtype must be substitutable ‚Äî avoid violating pre/post conditions.
  4. Interface Segregation: small cohesive interfaces.
  5. Dependency Inversion: depend on abstractions, use DI frameworks for complex systems.
üß† Collections: Patterns & Performance
  • Prefer ArrayList for read-mostly random access; LinkedList for frequent insert/removal at ends.
  • Use HashMap for average O(1) lookup; TreeMap for sorted keys.
  • For concurrency, use ConcurrentHashMap, CopyOnWriteArrayList, BlockingQueue.
  • Beware boxing/unboxing and prefer primitive-specialized libs for hot loops (Trove, fastutil) if needed.

public class InterruptDemo {
  public static void main(String[] args) throws InterruptedException {
    Thread t = new Thread(() -> {
      while (!Thread.currentThread().isInterrupted()) {
        // busy work
      }
      System.out.println("Stopped by interrupt");
    });
    t.start();
    Thread.sleep(50);
    t.interrupt();
  }
}

üß† Logical Coding Challenges

Practice with short problems and solutions; great for live coding rounds.

1. Print Pascal's triangle (n rows)
Use iterative combinatorics or build rows from previous row sums. Time O(n^2), space O(n).
2. Reverse words in a sentence (in-place)
Reverse whole string then reverse each word; O(n) time, O(1) extra space if char array.
3. Fibonacci: recursive vs dynamic programming
Recursive is exponential; use memoization or iterative DP to get O(n).
Sample solution ‚Äî Reverse words (code) 
public class ReverseWords {
  public static void main(String[] args){
    String s = "Hello World from Java";
    String[] parts = s.trim().split("\\s+");
    Collections.reverse(Arrays.asList(parts));
    System.out.println(String.join(" ", parts));
  }
}

üéØ HR + Technical Round Strategy

HR Round

  • Tell a concise story ‚Äî background, key projects, what you built, and what you learned.
  • Prepare answers for strengths, weaknesses (mention mitigation + learning), notice period, and salary expectations.
  • Ask thoughtful questions about team, mentorship, and learning opportunities.

Technical Round

  • Before coding: clarify requirements, edge cases, and input sizes; reason about complexity.
  • Write clean code, name variables clearly, and test on sample inputs; explain trade-offs.
  • Discuss follow-ups: optimizations, memory vs time trade-offs, concurrency considerations, and testing strategy.

Live Coding Tips

  • Start with a brute-force solution if stuck, then optimize; communicates problem-solving process.
  • Use small examples to validate logic; explain each step to the interviewer.
  • If stuck, ask clarifying questions ‚Äî interviewers prefer clarifying to guessing requirements.
