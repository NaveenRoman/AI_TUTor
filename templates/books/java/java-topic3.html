Defining the Data Structures

Array Definition

An **Array** is a linear data structure used to store a **fixed-size** sequential collection of elements of the **same data type**. It is a fundamental object in Java.

Key Usage: Brackets ([])

Use **square brackets** ([]) for declaration (int[] arr), initialization, and, crucially, for accessing elements via an **index** (e.g., myArray[2]).

String Definition

A **String** is a sequence of characters, used for text manipulation. In Java, it is treated as an object and is **immutable** (cannot be changed after creation).

Key Usage: Quotes & Methods

Defined using **double quotes** (e.g., "Hello"). Elements are accessed using methods like myString.charAt(i), **not** square brackets.

Array vs. String: Core Differences

Feature Array String
Content Holds any data type (int[], String[], etc.) Holds only character data (text)
Mutability **Mutable**: Elements can be changed after creation. **Immutable**: Cannot be changed (modification creates a new object).
Size Property **Fixed-size** (property .length). Variable (method .length()).
Access Index Brackets (arr[i]) Method access (str.charAt(i))

üì¶ Arrays in Java

In Java, an Array is a collection of similar data elements stored in contiguous memory locations. Each element is identified by an index, starting from 0. Arrays are useful when you need to store multiple values of the same type (e.g., integers, floats, strings) under a single variable name.

Syntax:

dataType[] arrayName = new dataType[size];  // Declaration with size
dataType[] arrayName = { val1, val2, val3 }; // Declaration with initialization

üìä Difference Between Arrays and Strings

Data Type Can store any primitive or object type. Stores only a sequence of characters.
Mutability Mutable (values can be changed). Immutable (once created, cannot be modified).
Declaration int[] arr = {1,2,3}; String s = "Hello";
Access Using index (e.g., arr[0]). Using methods (e.g., s.charAt(0)).
Length arr.length (property) s.length() (method)

3Ô∏è‚É£ java.util.Arrays Utility Class

Java provides a powerful utility class called Arrays that includes static methods for performing operations like sorting, searching, copying, and comparing arrays. 

import java.util.Arrays;

int[] numbers = {5, 3, 8, 1, 2};

// 1Ô∏è‚É£ Sort Array (ascending order)
Arrays.sort(numbers); // ‚Üí [1, 2, 3, 5, 8]

// 2Ô∏è‚É£ Binary Search (array must be sorted)
int idx = Arrays.binarySearch(numbers, 5); // ‚Üí 3

// 3Ô∏è‚É£ Copy Array
int[] copy = Arrays.copyOf(numbers, numbers.length);

// 4Ô∏è‚É£ Fill Array
int[] filled = new int[5];
Arrays.fill(filled, 7); // ‚Üí [7, 7, 7, 7, 7]

// 5Ô∏è‚É£ Compare Arrays
boolean same = Arrays.equals(numbers, copy);  // true
boolean ref = (numbers == copy);              // false
  • Arrays.sort() ‚Äî Sorts array in ascending order (in-place).
  • Arrays.binarySearch() ‚Äî Returns the index of an element (requires sorted array).
  • Arrays.copyOf() ‚Äî Creates a shallow copy (new memory).
  • Arrays.fill() ‚Äî Replaces all elements with a single value.
  • Arrays.equals() ‚Äî Checks element-by-element equality.
‚ñ∂ Show Output

Copied Array ‚Üí [1, 2, 3, 5, 8]

üîç Binary Search Result Interpretation

Note: The method Arrays.binarySearch() returns:

  • Index (0 or positive) ‚Üí if the key is found.
  • Negative value ‚Üí if the key is not found.

You can use this negative value to determine the insertion point ‚Äî where the element would appear if added to the array. 

// Assume the array is sorted
int[] numbers = {1, 2, 3, 5, 8};

// Searching for an element that doesn't exist
int pos = Arrays.binarySearch(numbers, 4); 

if (pos >= 0) {
    System.out.println("Found at index: " + pos);
} else {
    // Formula to find insertion position
    int insertPoint = -pos - 1;
    System.out.println("Not found. Insertion point: " + insertPoint);
}

Search Key = 4 ‚ùå (Not Found)
Position lies between 3 and 5 ‚Üí insertion index = 3

üëâ Insertion Point = -r - 1

üßµ Strings in Java

A String in Java is a sequence of characters enclosed in double quotes ("..."). Strings are immutable ‚Äî once created, their content cannot be changed. Java provides several utilities to manipulate, compare, and efficiently handle strings.

Definition: String is a class in java.lang package that represents a sequence of characters.

1Ô∏è‚É£ String Creation & Comparison

// String creation
String s1 = "Hello";              // Stored in String Pool
String s2 = new String("Hello");  // Created in Heap
String s3 = "Hello";              // Reuses same pool reference as s1

// Comparison examples
boolean refEq12 = (s1 == s2);      // false ‚Üí different objects
boolean refEq13 = (s1 == s3);      // true  ‚Üí same String Pool object
boolean valueEq = s1.equals(s2);   // true  ‚Üí content is same

System.out.println("s1 == s2 ? " + refEq12);
System.out.println("s1 == s3 ? " + refEq13);
System.out.println("s1.equals(s2)? " + valueEq);

// ‚òï Memory Representation
String Pool:  "Hello"  ‚Üê s1, s3 point here
Heap Object:  "Hello"  ‚Üê created separately for s2
  • String literals (e.g., "Hello") go into the String Pool.
  • Using new String("...") creates a new object on the heap.
  • == compares memory references, while .equals() compares actual content.

4Ô∏è‚É£ StringBuilder & StringBuffer ‚Äî Mutable Alternatives

Definition: StringBuilder and StringBuffer are mutable classes that allow in-place string modification without creating new objects. 
// 1Ô∏è‚É£ Using StringBuilder (fast, not thread-safe)
StringBuilder sb = new StringBuilder("Java");
sb.append(" Programming");     // adds text
sb.insert(4, " Language");     // inserts substring
sb.delete(0, 5);               // removes part
System.out.println(sb.toString());

// 2Ô∏è‚É£ Using StringBuffer (thread-safe)
StringBuffer sbuf = new StringBuffer("Hello");
sbuf.append(" World!");
System.out.println(sbuf.toString());
  • StringBuilder ‚Üí Non-synchronized (faster, single-threaded apps).
  • StringBuffer ‚Üí Synchronized (safe for multi-threaded apps).
  • Both modify content in-place, unlike immutable Strings.
  • Use .toString() when finished to get a regular String.

Character Handling ‚Äî Using the Character Class

The Character class is a **wrapper class** for primitive char values. It provides many **static helper methods** to classify, convert, and validate characters. 

// üß© Example: Character checks and conversions
char ch = 'A';

// Classification checks
boolean isLetter = Character.isLetter(ch);         // true ‚Äî 'A' is a letter
boolean isDigit  = Character.isDigit('9');         // true ‚Äî '9' is a digit
boolean isSpace  = Character.isWhitespace(' ');    // true ‚Äî space detected

// Case conversion
char lower = Character.toLowerCase('Z');           // 'z'
char upper = Character.toUpperCase('b');           // 'B'

// üßÆ Example: Counting digits in a String
String s = "Test123Test";
int digitCount = 0;
for (int i = 0; i < s.length(); i++) {
    char c = s.charAt(i);                          // extract each character
    if (Character.isDigit(c)) {                    // check if numeric
        digitCount++;
    }
}
System.out.println("Total Digits: " + digitCount);

üîç Code Breakdown

  • Character Wrapper: Allows you to use methods like isLetter() and toUpperCase() on a char.
  • Classification: Detect whether a char is a letter, digit, whitespace, etc.
  • Case Conversion: Change a single character‚Äôs case without affecting others.
  • Practical Use: Helpful for text parsing, validation, and pre-processing (like counting digits, spaces, or letters).

Cheat Sheet ‚Äî Key Rules to Remember

  • Arrays: Fixed-size, homogeneous data. Use arr.length (property) to get its size.
  • Strings: Immutable sequences of characters. Use str.length() (method). Never use == for comparison ‚Äî use .equals().
  • Performance: For multiple string edits (especially inside loops), use StringBuilder instead of String.
  • 2D Arrays: arr.length = number of rows; arr[i].length = number of columns in row i.
  • Character Utility: Use Character.isDigit(), isLetter(), toUpperCase() for robust text validation.
